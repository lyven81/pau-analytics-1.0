## 3B: Read the Template

Read these files from the selected category folder:
- `config.yaml` — business rules, services, pricing
- `user-guide.md` — what the system does, how it works

Extract:
- What the solution does (1 sentence)
- Who it helps (target business type)
- 2-3 specific capabilities (from config.yaml or user-guide.md)
- Demo link: `https://lyven81.github.io/pau-ai/[category].html`

---

## 3C: Write the Post

Write a LinkedIn post (150–250 words):

1. **Hook (lines 1–2):** A problem the target business faces daily. Frame it as pain, not as a feature.
   - Good: "Your receptionist missed 3 booking calls today. Each one was worth RM150."
   - Weak: "Introducing our new AI booking system."
2. **The solution (1 paragraph):** What the AI system does — in plain language, not tech jargon.
3. **Specific capability (1–2 sentences):** One concrete thing it handles (e.g., "It checks for conflicts, applies peak-hour pricing, and confirms the booking — all in one WhatsApp message.")
4. **Who uses it (1 sentence):** Name specific business types.
5. **CTA:** "See the demo: [demo URL]"
6. **Hashtags:** 3–5 relevant. Always include #AI #MalaysianSME.

Tone: practical, grounded. Never use "revolutionary", "cutting-edge", "game-changing". Write like you're explaining it to a kopitiam owner over coffee.
